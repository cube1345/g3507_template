## 2. 新 ops 设计原则

### Key 不适合 `device_switch_ops_t`
Key 是输入事件设备，不是开关设备。建议新增：

```c
typedef struct {
    uint8_t (*scan)(void *me);
    uint8_t (*read_level)(void *me);
} device_input_ops_t;
```

然后 Key 自己定义：

```c
typedef device_input_ops_t key_base_ops_t;
```

Key 的对象：

```c
typedef struct key_s {
    device_base_t base;
    const key_base_ops_t *ops;
    key_cfg_t cfg;
    Key_State state;
    uint32_t last_time;
    uint32_t continuous_time;
} key_t;
```

Key 的 board 层负责创建多个 key 对象，并封装不同 GPIO 极性。

---

### Encoder 也不适合 `device_switch_ops_t`
Encoder 是计数型传感器/输入设备。建议新增：

```c
typedef struct {
    int32_t (*get_count)(void *me);
    void (*reset_count)(void *me);
    void (*update_from_ab)(void *me, uint8_t changed_a, uint8_t changed_b, uint8_t level_a, uint8_t level_b);
} device_counter_ops_t;
```

然后 Encoder 自己定义：

```c
typedef device_counter_ops_t encoder_base_ops_t;
```

Encoder 对象：

```c
typedef struct encoder_s {
    device_base_t base;
    const encoder_base_ops_t *ops;
    encoder_cfg_t cfg;
    volatile int32_t count;
} encoder_t;
```

其中 `update_from_ab` 负责复用现有 A/B 相计数逻辑。

## 5. 业务层兼容策略

### Key
当前业务层主要通过 [`Key_GetValue`](MyUsed/MyLibs/OLED_menu.c:103) 使用。保持接口名不变即可。

### Encoder
当前 [`Mid/FreeRTOS_demo.c`](Mid/FreeRTOS_demo.c:211) 直接使用全局变量：

```c
extern int Get_Encoder_countA;
extern int Get_Encoder_countB;
```

短期建议保留全局变量兼容，避免大改业务层。

但新 facade 同时提供函数接口，后续可逐步迁移为：

```c
Encoder_GetCountA();
Encoder_ResetCountA();
```
